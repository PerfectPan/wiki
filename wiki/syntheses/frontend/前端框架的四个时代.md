# 前端框架的四个时代

## 问题

前端框架为什么会从早期模板和 MVC 风格，逐步走到组件化、再走到全栈 meta-framework？

## 简答

因为 Web 应用面对的问题一直在变：先是缺模块和构建，再是缺组件抽象和状态管理，后来又暴露出 SEO、性能、弱网和渐进增强的问题。每一代框架都在补前一代没有解决好的约束。

## 综合结论

- 这条 journal 已经给出了一个比较完整的历史分层：
  - 史前时代：没有成熟模块、构建和统一工程方法；
  - 第一代框架：Backbone / Angular 1 / Ember / Meteor 这类更完整的一站式应用框架；
  - 组件化视图层时代：React / Vue / Svelte 这类更聚焦渲染与组件生命周期的框架；
  - 全栈 meta-framework 时代：Next / Nuxt / Remix / Astro 等重新把路由、数据、部署和渐进增强拉回一体。
- 这说明所谓“框架演进”并不是线性替换，而是问题域在变化：
  - 早期先解决“如何组织应用”；
  - 中期解决“如何用组件抽象视图”；
  - 后期则重新面对 SSR、SEO、弱网、渐进增强和全栈协作问题。
- journal 里最值得保留的一条判断是：meta-framework 的回归，并不是回到第一代框架，而是建立在组件化与现代构建生态成熟之后，再把“原来必须自己拼”的东西重新纳入一站式方案。
