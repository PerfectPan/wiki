# 小程序架构演进

## 摘要

小程序架构演进的核心方向，是把浏览器内核、脚本执行、宿主能力和沙箱边界重新拆分，而不是只把 Web 页面塞进一个壳里。

## 关键点

- journal 里的记录说明，演进方向包括：
  - 通过 Chromium 做更深度定制；
  - 用 worker 化执行模型替代更原始的单线程心智；
  - 把 Node 能力和 JSAPI 以宿主受控方式暴露；
  - 通过封装 context 和接口边界强化沙箱能力。
- 这说明小程序运行时本质上是“受控宿主 + Web 技术栈”的组合，而不是浏览器的简单裁剪版。
