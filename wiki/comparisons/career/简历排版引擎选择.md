# 简历排版引擎选择

## 当前结论

如果目标是产出正式、可分页、对中英文混排要求较高的 PDF 简历，当前材料更支持优先考虑 `LaTeX`。`HTML/CSS`、`Typst`、`React-pdf` 各有可取之处，但在换行质量、分页和 CJK 支持同时成立这件事上，现阶段仍不如 `LaTeX` 稳。

## 备选项

- `HTML/CSS`：通用、灵活、实时预览友好。
- `LaTeX`：传统排版系统，强调排版质量和精确控制。
- `LaTeX.js`：把 LaTeX 思路带到浏览器环境。
- `Typst`：现代语法、编译快、体验更友好。
- `React-pdf`：以 React 组件方式生成 PDF。

## 取舍分析

| 方案 | 优势 | 风险 | 适用场景 |
| --- | --- | --- | --- |
| `HTML/CSS` | 通用性强，响应式和实时预览体验最好，开发门槛相对低。 | 原生分页弱，换行质量和 CJK 排版都需要额外补救，不适合把版面稳定性看得很重的正式简历。 | 更重视在线编辑体验，或同时需要 Web 页面和 PDF 输出。 |
| `LaTeX` | 换行质量高，分页成熟，CJK 支持较完整，适合追求最终 PDF 品质。 | 学习曲线陡，开发体验陈旧，实时预览和交互式编辑体验较差。 | 追求正式、稳定、可打印的高质量简历输出。 |
| `LaTeX.js` | 相比传统 `LaTeX` 更容易接近浏览器预览。 | 功能不完整，分页和换行质量不足，CJK 仍需额外处理。 | 想保留一点 LaTeX 风格，但又想靠近 Web 预览的实验性场景。 |
| `Typst` | 语法现代，编译快，可定制性好，比 `LaTeX` 更易上手。 | 生态较新，CJK 稳定性和实时预览能力仍有限。 | 愿意用较新的工具换开发体验，但能接受生态尚未完全成熟。 |
| `React-pdf` | React 集成自然，实时预览较友好，也支持分页。 | 排版能力和 CJK 支持仍有限，复杂版式控制不如专业排版系统。 | 已经在 React 技术栈内，希望复用组件思维生成 PDF。 |

## 推荐理由

1. `2024_12_24.md` 里最明确的判断，是简历排版的核心约束不只是“能不能生成 PDF”，而是换行质量、CJK、分页和编辑体验要如何排序。
2. 在这组约束下，`LaTeX` 虽然开发体验最差，但同时满足高质量换行、成熟分页和中日韩文字支持，因此当前更接近“正式简历默认选项”。
3. `Typst` 和 `React-pdf` 代表的是更现代、更易开发的方向，但从现有材料看，它们更像值得持续观察的候选，而不是已经完全替代 `LaTeX` 的结论。
4. `2023_03_15.md` 收集的简历模板也以 `moderncv`、`Awesome-CV`、Overleaf 模板为主，这和“先用成熟 LaTeX 生态交付结果”是一致的。
