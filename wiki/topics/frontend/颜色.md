# 颜色

- 色彩空间：https://ericportis.com/posts/2024/okay-color-spaces/
	- 这篇文章由 Eric Portis 撰写，标题为"Okay, Color Spaces"，发表于 2024 年 2 月 20 日。文章探讨了色彩空间的概念，解释了色彩空间是如何将颜色以某种方式排列在具有坐标的三维空间中。以下是文章的主要内容总结：
	- **色彩空间的定义**：色彩空间是一种将颜色按照特定方式排列在空间中的构造，这些空间是基于坐标系统构建的。
	- **颜色的几何关系**：文章指出，颜色之间并没有固有的几何关系，我们按照某种方式排列颜色是因为这样做对我们有用。
	- **CIE XYZ 色彩空间**：1931年，国际照明委员会（CIE）创建了 CIE XYZ 色彩空间，这是一个非常有用的色彩空间，它帮助我们预测不同颜色的混合结果。CIE XYZ 通过三个函数将光的波长转换为 X、Y、Z 三个坐标值，这些值定义了颜色在 CIE XYZ 空间中的位置。
	- **色彩空间的实用性**：CIE XYZ 色彩空间之所以有用，是因为它将颜色混合和匹配问题转化为数学问题。然而，它并不是感知上均匀的，这意味着在 CIE XYZ 空间中，颜色之间的感知差异并不均匀。
	- **感知均匀性**：文章讨论了感知均匀性的概念，即色彩空间中任意两点之间的距离与它们看起来的差异程度相对应。CIE XYZ 并不是感知均匀的色彩空间。
	- **Oklab 色彩空间**：文章介绍了 Oklab 色彩空间，这是一个较新的尝试，旨在创建一个感知上更均匀的色彩空间。Oklab 在数学和计算上简单，并且在大多数需要感知均匀性的任务中表现得比 CIE XYZ 和其他简单的三维色彩空间更好。
	- **OKLCH 色彩空间**：文章还提到了 OKLCH 色彩空间，这是 Oklab 的极坐标版本，它使用了与 Munsell 色彩系统相似的光度、色度和色调参数，使得颜色的感知属性更容易理解和操作。
	- **色彩空间的应用**：文章最后提到了一些实用工具，如 Evil Martians 的 OKLCH 色彩选择器和 Color.js库，这些工具可以帮助用户在实际应用中探索和使用不同的色彩空间。
	- 总的来说，文章强调了色彩空间是人为构造的概念，它们将颜色以有用的方式组织起来，尽管这些空间并不完美，但它们在数字成像和颜色管理中扮演着关键角色。
- **色域（Color Gamut）**：色域是指可以显示或打印的颜色范围。sRGB 是目前 Web 上广泛使用的标准，但新的标准支持如 Display P3 和 Rec2020 等更广的色域。
- **色彩空间（Color Space）**：色彩空间是基于特定色域标准构建的数学模型，用于描述色域中每个颜色的位置和关系。例如，sRGB 使用红、绿、蓝作为基础色，而 HSL 使用色相、饱和度和亮度。
	- 一种色域标准可以使用不同的色彩空间来描述，不同的色域标准也可以使用的是同一类的色彩空间表示。例如 sRGB 可以使用 `rgb()`、`hsl()`、`hwb()` 等方式进行描述，而像 Display-P3、Rec2020 色域都可以使用（R，G，B）的色彩空间来描述，只是空间的边界范围有所不同。
- **CSS Color 4标准**：新标准引入了 `color()` 函数和相关语法，允许开发者在 Web 页面上展示更广泛的色域和色彩空间。这意味着可以展示超出传统 sRGB 色域的色彩，从而提供更丰富的视觉体验。
