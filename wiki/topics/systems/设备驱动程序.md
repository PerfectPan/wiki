# 设备驱动程序

- I/O 设备的抽象：read / write / iosctl(设置设备的状态)
- 设备驱动程序就是把系统调用翻译成与设备寄存器的交互
	- 一段普通的内核代码
	- 但可能会睡眠
	- /dev/ 中的对象
		- /dev/null - null 设备
		- /dev/urandom - 随机数生成器设备
			- `head -c 512 [device] | xxd`
	- Linux 内核中最多也是质量最低的代码
- 可以理解设备驱动在 Liunx 环境下实现了 file_operations 的函数签名，Linunx 暴露了设备注册的方法，整体就插件化了
